## **一、前端工具與技術棧**

| 項目 | 說明 |
| :---- | :---- |
| **框架** | 靜態 HTML5 |
| **樣式** | Bootstrap v5.3.3 |
| **交互** | 原生 JavaScript (ES6+), fetch API |
| **狀態管理** | JWT Token (儲存在 localStorage) |

## 

## **二、介面規劃 (核心頁面)**

我們將採用 **單頁面應用 (SPA)** 的風格，只用一個 index.html，並用 JavaScript 來切換不同內容區塊的顯示，以模擬多頁面導航。

| \# | 頁面名稱 | 角色 | 核心功能 |
| :---- | :---- | :---- | :---- |
| **P1** | **登入/認證頁** | Public | 輸入帳號密碼，儲存並驗證 JWT Token。 |
| **P2** | **員工主頁 (訂餐)** | Employee | 顯示今日菜單（來自 /public/menu），選擇便當並呼叫 /orders 創建訂單。 |
| **P3** | **員工訂單記錄** | Employee | 顯示自己的歷史訂單（來自 /orders/mine），並提供 **刪除** 按鈕。 |
| **P4** | **總務主頁 (管理)** | Admin | 導航到 \[菜單管理\] 和 \[訂單統計\] 區域。 |
| **P5** | **總務：菜單管理** | Admin | 顯示餐廳和便當清單，提供新增/修改/刪除 (CRUD) 的 Bootstrap Modal 表單。 |
| **P6** | **總務：訂單統計** | Admin | 顯示當日訂單統計（來自 /orders/summary），顯示各個便當的總份數和總金額，並可進行 **結算** 操作。 |

## **三、手把手實作步驟流程**

### **Step 1: 基礎文件與環境設置**

創建前端目錄結構，並導入 Bootstrap 和 JavaScript 文件。

### **Step 2: 導航欄與區塊劃分**

使用 Bootstrap Navbar 建立導航欄，並設置不同的內容區塊 (\<section id="login-page"\>, \<section id="employee-page"\> 等)。

### **Step 3: 登入頁面 (P1) 實現**

設計簡潔的登入表單，並編寫 JavaScript 邏輯，處理表單提交並呼叫 /auth/login API。

### **Step 4: 狀態管理與導航**

編寫 JS 函式來儲存/讀取 JWT，並根據用戶權限動態顯示 Navbar 連結和內容區塊。

### **Step 5: 員工訂餐頁面 (P2, P3)**

實現菜單顯示和訂單創建的邏輯，以及個人歷史訂單的列表和刪除功能。

### **Step 6: 總務管理頁面 (P5, P6)**

使用 Modal 實現 CRUD 操作，並實現複雜的訂單統計數據展示。
